### Planning

I keep a list of blog posts to write. I use Google Keep on my phone to quickly capture new ideas. There is another list on Evernote which I wrote before getting serious about the blog. I will export one title per week to 'This Week' list on Trello.

### The first dirty draft

I use Sublime Text or VS Code to write the post. I don't want any spell or grammar checks while I am writing. MS Word and Grammarly make real-time checks. I treat that like writing obstacles. Especially when writing the dirty first draft. I write whatever gets into my mind. I even don't care about spellings. If I feel the post is going to be a long one, I will divide the post into the topics that I want to cover as headings and fill the content below the headings. Lengthly posts need a logical sequence for clarity and for easier navigation to the reader.

### Give some time

I get back to the writing after a day to see whether what I wrote is still valid for the context and scope. This is where I do my first edits. Giving some time to the idea will ground the excitement and refines the writing.

### Grammarly

When I am done editing, I will export the text to Grammarly for spell and grammar check. I like word suggestions feature in the editor and I heavily use it. Once everything is solid, I will paste the text back in my code editor.

### Push the changes

I keep the tags relevant to the post and push to the remote. This blog is hosted on Netlify via GitHub. Whenever I push to master, webhooks take control and Netlify grabs the latest commit and builds the website again. Such a nice service.

### Read again

After publishing it, I will read it again on the website. Twice I found some inconsistencies in the sentences which I and Grammarly also overlooked. Either I will rectify them and republish the changes from the code editor or I will directly edit on GitHub and pull the changes down.

And I will go to the next title.
